# ShopInsight 仓库级 Agent 规则

## 沟通语言

- 除非用户明确要求使用英文，否则所有沟通、讲解、学习笔记和交付说明均使用中文。

## 项目学习任务必须使用的技能

当用户要求仓库导览、项目学习、学习计划、代码链路追踪、面试准备或继续某个学习日程时，必须按以下顺序使用两个技能：

1. `orient:orient`
   - 先阅读当前源码、`AGENTS.md`、`README.md`、项目级 `.codex/skills/learning-opportunities/resources/orientation.md`、已有学习笔记和相关测试。
   - 所有结论以当前源码为准；README 和历史文档仅用于对照，不得代替源码取证。
   - 已存在且仍适用的 `orientation.md` 应作为学习基线，避免无意义地重讲整个项目；代码发生实质变化时才重新生成或更新。

2. `learning-opportunities:learning-opportunities`
   - 使用交互式学习方式，一次只进行一个问题或一个练习，提出问题后必须等待用户回答。
   - 按真实请求或数据调用顺序推进，不按目录逐文件泛读。
   - 每个小模块遵循：源码路径、类名和方法名定位 → 用户自行阅读并复述 → 根据回答纠正 → 再进入下一模块。
   - 用户回答错误时先提供最小提示，引导其回到源码定位；回答正确后再用异常路径、State 变化、测试或设计取舍验证理解。
   - 不提前一次性给出完整学习内容，不用长篇答案替代用户主动复述。

## 学习连续性

- 开始新的学习日程前，先读取之前生成的导览、学习笔记、缺陷记录和学习日志，明确已经掌握的内容与本次衔接位置。
- 不重复讲解用户已经掌握的模块，除非通过简短检索练习发现理解缺口。
- 学习过程中重点追踪：调用入口、关键函数、State/Context 变化、外部依赖边界、条件分支、异常路径和面试设计取舍。
- 每个学习日结束时生成对应学习笔记，至少包含：
  - 完整调用链；
  - 核心类与方法；
  - 一份可口述的面试讲解；
  - 高频面试问题及追问链；
  - 用户尚未掌握的薄弱点。
- 除非用户明确要求实现或验证理解确有必要，不得在学习任务中修改项目业务代码。
